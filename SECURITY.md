# Security

## Reporting a vulnerability

Please report security issues **privately**. Do not open a public issue.

- **Email:** Use the contact method listed in the repository (e.g. maintainer email or GitHub security advisory).
- **What to include:** Description of the issue, steps to reproduce, and impact if known.

We will acknowledge and respond as soon as possible.

## Scope

- Protocol contracts, canonical serialization, registry behavior, and settlement logic.
- Out of scope: general dependency vulnerabilities (report to the upstream project) or deployment/infra configuration outside this repo.
